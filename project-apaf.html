APAF

Tor Hidden Services allow people to access and share information securely and anonymously. This very important for people that are interested in making their voice be heard, but without having to worry about the possible repercussion that could come from being easily identifiable.
For example a political dissident may be interested in expressing his point of view, but is worried that by doing so he will face physical harm.
Up until now Tor Hidden Services required some basic technical skills to set them up, while setting up server applications implied an high degree of complexity for non-technically proficient users.
The goal of APAF is reducing the technical requirements for building and setting up a desktop oriented server side anonymous applications running as a Tor hidden service.

The main elements characterizing APAF are:

  • Enables easy / secure development of Anonymous Server applications
  • Reduces the complexity to run Anonymous Server Application (lower entrance barrier)
  • Provides a secure environment to develop Python applications bundling Tor

Key points

Below some of the main enabling key points of APAF

  • Anonymous Publishing
  • Designed to build distributed network of anonymous services
  • Practical for Tactical Use (low resources, low requirements)
  • Increases Outreach of Anonymity software reducing complexity for server setup
  • Reduces complexity to in cross-platform software development and Anonymity integration
  • Supports creation of a wide network of Services running within Tor Darknet
  • Easy Integration of security tools invisible for the users

The Software

APAF, or the Anonymous Python Application Framework, aims at creating a build environment and framework to allow Python based server side applications to be shipped to desktop users.

The main platforms where APAF applications can be delivered and securely deployed are:

  • Windows x86
  • Mac OS X x86
  • Ubuntu Desktop 12.04
  • Debian 5.0/6.0 (server)

A developer will build a Python application with APAF and then deliver to the end user a single executable that allows them to start such a service.
The end goal will be to provide a set of server applications built with APAF that users will be able to download and use. Think for example of the blog.exe software that a user can run and instantly start publishing their anonymous blog.
Because we are using the Tor Hidden Service technology the user of the software is not required to have a static IP address, register a domain name, nor have to make changes to their router configuration.
If the server exposed by the user is delivered via web (HTTP), then it can also be internet exposed via services such as Tor2web (LINK A pagina Tor2web).
